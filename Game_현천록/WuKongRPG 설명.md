# CHSportfolio

## 게임명 : 현천록 (소울라이크 액션 RPG)

### 프로젝트 개요
- **개발 기간:** 2025.9 ~ 2025.12
- **개발 인원:** 5명 (보스 전투 담당 1, 아트 및 이펙트 담당 1, 몬스터 전투 담당 1, 유저 애니메이션 담당 1, UI 및 게임 시스템 구현, 프로젝트 관리 1) / UI, 게임 시스템 구현, 프로젝트 관리 담당
- **사용 기술 및 환경:** Unreal Engine 5, Blueprint 사용
- **장르:** 캐주얼 액션 RPG

### 주요 구현 내용

#### 1. 저스트 회피 시스템 구현
- **초산 공간 개념 차용:** "퍼니싱 그레이 레이븐"의 초산 공간 시스템을 차용하여, 저스트 회피 성공 시 적에게 슬로우 모션을 부여하고 화면 전체를 흑백으로 전환하는 포스트 프로세싱 효과를 추가하였습니다.
- **전투 경험 강화:** 유저에게 정확한 타이밍의 회피를 통한 명확한 이점을 부여하여 액션성을 추가하고, 타이밍에 맞게 회피하여 이득을 가져간다는 재미를 더했습니다.

**관련 Blueprints**
- **JustEvadeBP:** 저스트 회피 판정 성공 시 트리거됩니다.
- **시간 지연 및 시각 효과 제어:** `Set Global Time Dilation` 노드를 사용하여 게임 전체의 시간을 느리게(예: 0.2배속) 만들고, 플레이어 캐릭터의 `Custom Time Dilation`을 높여(예: 5.0배속) 플레이어만 정상 속도로 움직일 수 있도록 구현했습니다. 
- **포스트 프로세싱:** `Make Struct PostProcessSettings` 노드를 통해 화면의 `Saturation`(채도) 값을 0으로 낮춰 흑백 화면을 연출하고, 일정 시간(Delay)이 지나면 다시 원래의 시간 속도와 색상으로 복구합니다.
- 

#### 2. 스태미나 시스템 구현
소울라이크 장르의 핵심인 자원 관리 요소를 위한 스태미나 시스템을 구현했습니다.
- **자원 소모 및 회복:** 회피, 달리기 등 특정 액션 수행 시 스태미나가 감소하며, 액션을 취하지 않을 때는 자연적으로 회복되도록 상태 기반 로직을 구성했습니다.
- **UI 연출:** 스태미나와 체력의 증감을 UI에 반영할 때, 수치가 즉각적으로 변하는 것이 아니라 FInterp To 노드를 사용하여 부드럽게 줄어들거나 차오르는 보간 이펙트를 구현했습니다. 이를 통해 몬스터 헌터나 다크 소울 시리즈 등에서 볼 수 있는 시각적으로 인지하기 쉽고 자연스러운 체력, 스태미너 바 UI 연출을 구현했습니다. 

**관련 Blueprints**
- **StaminaBP:** 매 프레임마다 플레이어의 상태(달리기, 회피 등)를 체크하여 스태미나 값을 실시간으로 처리합니다.
- **행동 제한 로직:** 스태미나가 0 이하로 떨어졌을 때 달리기, 점프 등의 특정 행동을 제한하는 Branch 로직을 통해 스태미너가 있을 때에만 해당 행동을 할 수 있도록 구현하였습니다.
- **UI 보간 처리:** Displayed Stamina 및 Displayed Hp 변수를 별도로 두어, 실제 스탯 값과 목표 값 사이를 FInterp To로 부드럽게 연결하여 화면에 출력되도록 처리했습니다.

#### 3. UI 및 Scene 구현
게임의 전반적인 흐름을 제어하는 다양한 UI 위젯과 시스템을 구축했습니다. (Scene Image 참조)
- **UI 위젯 제작:** 메인 메뉴, 로딩 화면, 일시 정지, 사망 화면, 엔딩 씬 등 게임 진행에 필요한 모든 주요 UI를 UMG를 통해 구현했습니다.
- **세이브 포인트 시스템:** 맵 곳곳에 배치된 세이브 포인트 오브젝트와 상호작용 시, 플레이어의 체력을 회복하고 해당 위치를 부활 지점으로 GameInstance에 저장하는 로직을 구현했습니다. 상호작용 시 화면 중앙에 나타나는 세이브 완료 UI와 사운드 이펙트를 추가하였습니다.

#### 4. 애니메이션 Notify를 활용한 Collision 및 콤보 시스템 구현
- **Collision 제어:** 공격 애니메이션 진행 시 무기에 항상 Collision이 켜져 있는 문제를 방지하기 위해, 타격 모션 시작 시점에만 Notify(AnimNotify_StartAttack)를 활성화하여 Collision이 켜지도록 구현했습니다. 타격이 끝나면 AnimNotify_EnableAttack을 통해 Collision을 끕니다.
- **콤보 애니메이션 흐름 제어:** 콤보 공격(1단, 2단, 3단) 진행 중 다음 공격으로 이어질 수 있는 유예 시간이 끝날 경우, AnimNotify_ResetCombo를 활성화하여 콤보 상태를 원점으로 되돌리도록 구현했습니다.

**관련 Blueprints**
- **AnimNotifyBP:** StartAttack, EndAttack Notify Event를 통하여 Collision의 활성화및 비활성화를 관리하고, ResetCombo를 통해 콤보시스템에서 단계 초기화를 처리했습니다.

#### 5. 프로젝트 및 버전 관리 (Microsoft Azure)
- **Azure DevOps 환경 구축:** 팀 프로젝트의 원활한 협업을 위해 Microsoft Azure 환경을 구축하고 프로젝트의 전체적인 버전 관리를 수행했습니다.
- **통합 및 충돌 해결 전담:** 여러 파트의 작업물을 병합하는 과정에서 발생하는 블루프린트 충돌, 레벨 데이터 유실 등의 통합 이슈를 전담하여 처리했습니다. 버전 롤백 및 복구 작업을 통해 프로젝트 지연 및 오류를 방지했습니다.

### 트러블 슈팅

**1. 협업 환경에서의 레벨 및 데이터 유실 해결**
* **문제 상황:** 아트, 보스 전투, 메인 전투 파트 병합 과정 중, 버전 최신화가 누락된 상태로 push되어 이후 작업 데이터가 유실되는 문제가 발생했습니다.
* **해결 과정:** Azure DevOps 히스토리를 추적하여 병합 이전 버전으로 롤백 후 재병합을 수행해 프로젝트 지연을 방지했습니다.
* **시스템 개선:** 이후 동일한 문제 발생을 방지하기 위해, Discord의 Webhook 시스템을 Azure 레포지토리와 연동하여 업데이트 내역을 팀원 모두가 실시간으로 공유하고 확인할 수 있는 환경을 구축했습니다.

**2. 저스트 회피(JustEvade) 판정 타이밍 개선**
* **문제 상황:** 몬스터의 실제 타격이 시작되기도 전에 회피 판정이 발생하여, JustEvade가 지나치게 쉽게 발동되는 밸런스 문제가 있었습니다.
* **해결 과정:** 몬스터 공격 몽타주에 ANS_JustEvade (AnimNotifyState) 트랙을 추가하고, 해당 구간이 활성화된 상태에서만 JustEvade 판정이 이루어지도록 로직을 수정했습니다.
* **디테일 조정:** 판정 구간을 무기의 실제 Collision 유지 시간과 완전히 동일하게 맞출 경우 회피 난이도가 너무 어려워지는 문제가 발생했고, 
이를 해결하기 위해 Collision 활성화 직전의 프레임에도 약간의 버퍼을 부여하여 난이도를 조정하였습니다.

### 프로젝트 회고

**기존 프로젝트 대비 개선점**
* 이전 프로젝트(Gladiator Game)에서 시스템 결합을 프로젝트 막바지에 진행하여 통합에 실패했던 한계를 개선했습니다.
* 본 프로젝트에서는 전체 개발 후 일괄 통합하는 방식이 아닌, 부분적인 시스템 개발이 완료될 때마다 즉시 통합 테스트를 진행하는 방식으로 프로세스를 변경하여 병합 충돌을 최소화하고 최종 빌드에 성공했습니다.

**한계점 및 아쉬운 점**
* 시간 부족으로 인해 각 몬스터의 공격에 따른 세부적인 데미지 기믹 구현을 완료하지 못했습니다.
* Git LFS(Large File Storage) 포인터 이슈로 인한 파일 유실 문제가 발생하여 해결을 위해 전체적인 개발 시간이 지연되었습니다. 
* Tool 숙련도 부족으로 인한 팀원 간 Azure DevOps 권한 부여 누락 문제가 존재했습니다.
* JustEvade 판정 로직을 노티파이로 개선했으나, 몬스터별 애니메이션 프레임 차이로 인해 해당 문제를 완벽하게 개선하지는 못했습니다.

**배운 점**
* 대용량 에셋을 다루는 게임프로젝트 개발환경에서는 Git LFS 포인터 관리에 대한 원리를 정확히 파악하고 주의를 기울여야 함을 배웠습니다.
* 원활한 협업을 위해 GitHub뿐만 아니라 Azure DevOps 등 다양한 버전 관리 툴에 대한 지식의 필요성을 체감하였습니다.
